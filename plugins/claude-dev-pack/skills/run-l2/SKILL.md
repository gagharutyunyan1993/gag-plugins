---
description: Two improvement iterations — first for correctness, second for code quality
---

Improve the result in two focused passes.

Pass 1 — Correctness:
- Fix logic errors
- Handle missing edge cases
- Verify data flow
- Ensure tests cover critical paths

Pass 2 — Code quality:
- Improve naming and structure
- Reduce unnecessary complexity
- Improve readability
- Ensure consistency with codebase conventions

Rules:
- Each pass must materially improve the previous one
- Focus on material improvements only
- Stop if no material improvement is possible

Output format:
1. Pass 1: correctness findings and fixes
2. Pass 2: quality improvements
3. Final improved version
